4 Lenses🔗

This section documents the lens system — a composable, purely functional approach to focusing on and updating values inside nested data structures.

4.1 🔗

4.1.1 Core Lens🔗

 (require freer-lib/lens/lens) package: freer-lib

This module provides the core lens abstraction — a composable pair of get and set functions that enables focused, immutable access into nested data structures.

struct

(struct lens (get set)
    #:extra-constructor-name make-lens
    #:transparent)
  get : procedure?
  set : procedure?
A first-class lens value pairing a getter and a setter.

  • get: A procedure (-> structure? any/c) that extracts a focused value from a structure.

  • set: A procedure (-> any/c structure? structure?) that returns a new structure with the focused value replaced.

Examples:
> (define first-lens
    (lens first
          (lambda (new-v lst) (list-set lst 0 new-v))))

first: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> first-lens

first-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

procedure

(view-l l structure) → any/c

  l : lens?
  structure : any/c
Extracts the focused value from structure using the lens l.

Example:
> (view-l first-lens '(10 20 30))

first-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

procedure

(set-l l new-value structure) → any/c

  l : lens?
  new-value : any/c
  structure : any/c
Returns a new structure with the focused position updated to new-value.

Example:
> (set-l first-lens 99 '(10 20 30))

first-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

syntax

(set-l* structure [l new-v] ...)

A syntactic shorthand for chaining multiple set operations on a single structure. Each [l new-v] clause is applied left-to-right. Returns structure unchanged when no clauses are provided.

Examples:
> (define second-lens
    (lens second
          (lambda (new-v lst) (list-set lst 1 new-v))))

second: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> (set-l* '(1 2 3)
        [first-lens 10]
        [second-lens 20])

second-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

procedure

(over-l l update-fn structure) → any/c

  l : lens?
  update-fn : procedure?
  structure : any/c
Applies update-fn to the focused value inside structure and returns the updated structure. Equivalent to (set-l l (update-fn (view-l l structure)) structure).

Example:
> (over-l first-lens add1 '(10 20 30))

first-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

syntax

(lens-compose l1 l2 ...)

Composes two or more lenses left-to-right, where l1 focuses inside the target of l2. The resulting lens focuses on the value that l1 would see after l2 has zoomed in.

Examples:
> (define nested '((1 2) (3 4)))
> (define inner-first-of-second
    (lens-compose first-lens second-lens))

first-lens: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> (view-l inner-first-of-second nested)

inner-first-of-second: undefined;

 cannot reference an identifier before its definition

  in module: top-level

> (set-l  inner-first-of-second 99 nested)

inner-first-of-second: undefined;

 cannot reference an identifier before its definition

  in module: top-level

4.2 🔗

4.2.1 Common Lenses🔗

 (require freer-lib/lens/common-lenses) package: freer-lib

This module provides ready-made lenses for common Racket data structures, built on top of freer-lib/lens/lens.

procedure

(hash-lens key) → lens?

  key : any/c
Returns a lens that focuses on the value associated with key in a hash table. The get side uses hash-ref and the set side uses hash-set, producing a new hash rather than mutating the original.

Examples:
> (define db (hash 'name "Alice" 'age 30))
> (view-l (hash-lens 'name) db)

"Alice"

> (set-l  (hash-lens 'age)  31 db)

'#hash((age . 31) (name . "Alice"))

> (over-l (hash-lens 'age)  add1 db)

'#hash((age . 31) (name . "Alice"))

syntax

(struct-lens id field-id)

Constructs a lens that focuses on field-id within an instance of the struct id. The accessor id-field-id is used for get, and struct-copy is used for set so the original instance is never mutated.

Examples:
> (struct point (x y) #:transparent)
> (define point-x (struct-lens point x))
> (define point-y (struct-lens point y))
> (define p (point 3 7))
> (view-l point-x p)

3

> (set-l  point-y 42 p)

application: not a procedure;

 expected a procedure that can be applied to arguments

  given: (lens #<procedure:point-x>

#<procedure:...s/common-lenses.rkt:21:10>)

> (over-l point-x add1 p)

4.3 🔗

4.3.1 Struct Lenses🔗

 (require freer-lib/lens/struct-lens) package: freer-lib

This module provides struct/lens, a drop-in replacement for struct that automatically generates a lens for every declared field, and struct/lens-out, a provide-form that exports the struct together with all its generated lenses.

syntax

(struct/lens name (field ...) option ...)

Defines a struct named name with the given fields, exactly as struct would, then defines one lens per field. Each lens is named name->field.

Examples:
> (struct/lens person (name age email) #:transparent)
> (define alice (person "Alice" 30 "alice@example.com"))
> (view-l person->name  alice)

"Alice"

> (set-l  person->age   31 alice)

(person "Alice" 31 "alice@example.com")

> (over-l person->email string-upcase alice)

(person "Alice" 30 "ALICE@EXAMPLE.COM")

Lenses compose naturally with lens-compose for working inside nested structures:

Examples:
> (struct/lens team (lead) #:transparent)
> (define t (team alice))
> (define team-lead-name (lens-compose person->name team->lead))
> (view-l team-lead-name t)

person-name: contract violation

  expected: person?

  given: (team (person "Alice" 30 "alice@example.com"))

> (set-l  team-lead-name "Bob" t)

person-name: contract violation

  expected: person?

  given: (team (person "Alice" 30 "alice@example.com"))

syntax

(struct/lens-out struct-type)

A provide form that exports struct-type via struct-out and additionally exports every lens generated by struct/lens for that struct type.

Examples:
> (module persons freer-lib/struct-lens
    (provide (struct/lens-out person))
    (struct/lens person (name age) #:transparent))
