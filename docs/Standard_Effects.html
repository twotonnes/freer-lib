3 Standard EffectsðŸ”—

While the core library allows you to define any effect you wish, this module provides a set of common, pre-defined effects for everyday tasks like Input/Output, HTTP requests, system commands, error handling, and logging.

These effects come with companion **default handlers**. You are not required to use these handlers—you can write your own interpreters for cmd-effect, http-effect, fail-effect, or log-effect if you wish to mock them for testing—but the default handlers provide production-ready implementations.

3.1 Understanding Basic EffectsðŸ”—

To understand how these standard effects work, consider the simplest one: the Identity effect.

 (require freer-lib/effects/id-effect) package: freer-lib

struct

(struct id-effect (value)
    #:extra-constructor-name make-id-effect
    #:transparent)
  value : any/c
A descriptor that simply holds a value.

procedure

(id v) → impure?

  v : any/c
Constructs an effect that wraps v.

When you use id, you are effectively pausing the computation to hand a value to the handler, which is expected to hand it right back.

Examples:
> (define computation
    (do [x <- (id 10)]
        [y <- (id 20)]
        (return (+ x y))))
> (with-impure-handlers
    ([(id-effect v) (return v)])
    computation)

3.2 System CommandsðŸ”—

 (require freer-lib/effects/cmd-effect) package: freer-lib

This module provides effects for executing shell commands and capturing their output.

struct

(struct cmd-effect (command)
    #:extra-constructor-name make-cmd-effect
    #:transparent)
  command : string?
An effect descriptor for running a system command.

struct

(struct cmd-result (out err exit-code)
    #:extra-constructor-name make-cmd-result
    #:transparent)
  out : string?
  err : string?
  exit-code : byte?
The structure returned by the handler after a command completes successfully.

  • out: Standard output captured as a string.

  • err: Standard error captured as a string.

  • exit-code: The integer exit code of the process.

procedure

(cmd command) → impure?

  command : string?
Creates an effect that requests the execution of command.

Example:
> (define (check-status)
    (do [res <- (cmd "echo \"checking git status\"")]
        (if (zero? (cmd-result-exit-code res))
            (return (cmd-result-out res))
            (return "Git check failed"))))

procedure

(execute-command command) → (or/c pure? impure?)

  command : string?
The default implementation for executing commands. It attempts to run the command string using the system’s shell (specifically targeting PowerShell on Windows environments).

  • Success: Returns a pure value containing a cmd-result.

  • Failure: If a Racket exception occurs during execution (such as the shell not being found), this function catches the exception and uses the fail effect to propagate the error message.

Error Handling: When execute-command encounters an error (e.g., shell not found, permission denied), it uses the fail-effect to signal the error. Your computation handler must be prepared to handle the fail-effect.

Example:
> (run (check-status)
       (lambda (eff k)
         (match eff
           [(cmd-effect c) (k (execute-command c))]
           [(fail-effect msg)
            (displayln (format "Command error: ~a" msg) (current-error-port))
            (return (void))])))

3.3 HTTP RequestsðŸ”—

 (require freer-lib/effects/http-effect) package: freer-lib

This module provides effects for performing HTTP network requests.

struct

(struct http-effect (url method headers body)
    #:extra-constructor-name make-http-effect
    #:transparent)
  url : url?
  method : symbol?
  headers : (listof string?)
  body : (or/c #f bytes? string?)
The descriptor containing all necessary information to perform a request.

procedure

(http-get url headers) → impure?

  url : string?
  headers : (listof string?)
Creates a GET request effect.

Example:
> (define (fetch-home)
    (do [(list status headers body) <- (http-get "https://racket-lang.org" '())]
        (return (format "Status: ~a, Body length: ~a"
                        status
                        (string-length body)))))

procedure

(http-post url headers body) → impure?

  url : string?
  headers : (listof string?)
  body : (or/c bytes? string?)
Creates a POST request effect.

procedure

(http-put url headers body) → impure?

  url : string?
  headers : (listof string?)
  body : (or/c bytes? string?)
Creates a PUT request effect.

procedure

(http-patch url headers body) → impure?

  url : string?
  headers : (listof string?)
  body : (or/c bytes? string?)
Creates a PATCH request effect.

procedure

(http-delete url headers) → impure?

  url : string?
  headers : (listof string?)
Creates a DELETE request effect.

procedure

(perform-http-request eff) → impure?

  eff : http-effect?
The default handler implementation using Racket’s net/url library.

It resumes the computation with a list containing:

'(status-code headers-list body-string)

  • status-code: Exact integer (e.g., 200, 404).

  • headers-list: A list of strings representing response headers.

  • body-string: The response body.

Error Handling: If an exception occurs during the HTTP request (e.g., network error, malformed URL, connection timeout), perform-http-request catches the exception and uses the fail effect to propagate the error message. This allows network errors to be handled gracefully within your computation.

Example:
> (run (fetch-home)
       (lambda (eff k)
         (match eff
           [(http-effect url method headers body)
  
            (k (list 200 '("Content-Type: text/html") "<html>Mock Body</html>"))]
           [(fail-effect msg)
  
            (displayln (format "Network error: ~a" msg) (current-error-port))
            (return (list 0 '() ""))])))

3.4 The Nothing EffectðŸ”—

 (require freer-lib/effects/nothing-effect)
  package: freer-lib

struct

(struct nothing-effect ()
    #:extra-constructor-name make-nothing-effect
    #:transparent)
An effect descriptor representing the absence of a value or a specific "no-op" signal.

procedure

(nothing) → impure?

Creates a nothing effect.

Examples:
> (define (safe-div x y)
    (if (zero? y)
        (nothing)
        (return (/ x y))))
> (define (calc-ratio a b)
    (do [res <- (safe-div a b)]
        (return (format "Ratio is ~a" res))))

This effect is often used by handlers of other effects to signal failure or early termination without crashing the entire program, effectively replacing a typed result with "nothing".

When handling this effect, it is common to either abort the computation or resume with a default value (like (void) or #f).

Examples:
> (with-impure-handlers
    ([(nothing-effect) (return 0.0)])
    (calc-ratio 10 0))

3.5 Fail EffectðŸ”—

 (require freer-lib/effects/fail-effect) package: freer-lib

The fail effect allows computations to signal failure with an error message. This is useful for error handling and control flow in effectful computations.

struct

(struct fail-effect (msg)
    #:extra-constructor-name make-fail-effect
    #:transparent)
  msg : string?
A descriptor that represents a failure with an associated error message.

procedure

(fail msg) → free?

  msg : string?
Constructs an effect that signals failure with the given error message.

The fail effect is often used in conjunction with error handling in other effects. When an error occurs during command execution or HTTP requests, the fail effect can be used to propagate the error through the computation.

Example:
> (define computation
    (do [x <- (id 10)]
        (if (< x 20)
            (fail "Value is too small")
            (return x))))

3.6 LoggingðŸ”—

 (require freer-lib/effects/log-effect) package: freer-lib

This module provides effects for logging messages at different severity levels.

struct

(struct log-effect (level message)
    #:extra-constructor-name make-log-effect
    #:transparent)
  level : symbol?
  message : string?
An effect descriptor for logging a message at a specified level.

  • level: A symbol indicating the log level (e.g., 'DEBUG, 'INFO, 'ERROR).

  • message: The log message as a string.

procedure

(log-debug message args ...) → free?

  message : string?
  args : any/c
Creates a logging effect that logs a message at the DEBUG level. Supports format strings with additional arguments for formatting.

Example:
> (define (process-data)
    (do [_ <- (log-debug "Starting data processing")]
        [result <- (return 42)]
        [_ <- (log-debug "Processing complete with result: ~a" result)]
        (return result)))

procedure

(log-info message args ...) → free?

  message : string?
  args : any/c
Creates a logging effect that logs a message at the INFO level. Supports format strings with additional arguments for formatting.

Example:
> (define (startup)
    (do [_ <- (log-info "Application started")]
        (return (void))))

procedure

(log-error message args ...) → free?

  message : string?
  args : any/c
Creates a logging effect that logs a message at the ERROR level. Supports format strings with additional arguments for formatting.

Example:
> (define (handle-failure)
    (do [_ <- (log-error "Critical failure occurred")]
        (return (void))))

procedure

(default-log-display eff) → void?

  eff : log-effect?
The default handler for displaying log messages. Routes ERROR level messages to current-error-port and other levels to current-output-port. Automatically formats messages using default-log-format and flushes the output.

procedure

(default-log-format eff) → string?

  eff : log-effect?
The default formatter for log messages. Formats the log entry with the current date (in ISO 8601 format), log level, and message as a string.

procedure

(write-log eff [display-proc format-proc]) → free?

  eff : log-effect?
  display-proc : (-> log-effect? void?) = default-log-display
  format-proc : (-> log-effect? string?) = default-log-format
Writes a log message using customizable display and formatting functions.

  • eff: The log effect to write.

  • display-proc: A function that handles displaying the formatted log message (default: default-log-display).

  • format-proc: A function that formats the log effect into a string (default: default-log-format).

If an exception occurs during logging, this function uses the fail-effect to propagate the error message.

Examples:
> (define (simple-format eff)
    (format "[~a] ~a" (log-effect-level eff) (log-effect-message eff)))
> (run (log-info "Custom logging example")
       (lambda (eff k)
         (match eff
           [(log-effect level msg)
            (k (write-log eff default-log-display simple-format))]
           [(fail-effect msg)
            (displayln (format "Logging failed: ~a" msg))
            (return (void))])))

Error Handling: The log effect is typically used for observability and diagnostics. It does not produce errors itself, but can be combined with other effects. When logging is not available (e.g., in a testing environment), you can write a handler that ignores log messages or collects them for inspection.

Example:
> (define (run-with-logging computation)
    (run computation
         (lambda (eff k)
           (match eff
             [(log-effect level msg)
              (k (write-log eff))]))))
