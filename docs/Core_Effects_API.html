1 Core Effects APIðŸ”—

The core API defines the structure of effectful computations and the machinery to execute them.

struct

(struct effect-desc ()
    #:extra-constructor-name make-effect-desc
    #:transparent)
The base structure for all effect descriptions. Users should define specific effects by creating subtypes of effect-desc. These descriptors hold the metadata required by a handler to fulfill an effect request.

Examples:
> (struct kv-get effect-desc (key) #:transparent)
> (kv-get "username")

(kv-get "username")

struct

(struct pure (value)
    #:extra-constructor-name make-pure
    #:transparent)
  value : any/c
Represents a computation that has completed successfully with a result value. This is the base case of the free monad.

struct

(struct effect (description k)
    #:extra-constructor-name make-effect
    #:transparent)
  description : effect-desc?
  k : (-> any/c any/c)
Represents a suspended computation waiting on an effect.

  • description: The payload describing the requested effect.

  • k: The continuation function. It accepts the result of the effect and returns the next step of the computation.

procedure

(return v) → pure?

  v : any/c
Lifts a raw value v into the effect context. This is equivalent to constructing (pure v).

Example:
> (return 42)

(pure 42)

procedure

(>>= m f) → (or/c pure? effect?)

  m : (or/c pure? effect?)
  f : (-> any/c (or/c pure? effect?))
Sequences two effectful computations. It applies the function f to the result of m once m produces a value.

If m is a pure value, f is applied immediately. If m is an effect, the binding is pushed down into the continuation k, creating a new computation that pauses for the same effect but runs f after the original continuation completes.

Examples:
> (struct increment effect-desc ())
> (>>= (return 10)
       (lambda (x)
         (effect (increment) (lambda (res) (return (+ x res))))))

(effect (increment ...) #<procedure>)

procedure

(run m handle) → any/c

  m : (or/c pure? effect?)
  handle : (-> effect? (or/c pure? effect?))
Executes an effectful computation m by repeatedly resolving effects using the provided handle function.

When run encounters a pure value, it returns the unwrapped value. When it encounters an effect, it passes the effect object to handle. The handler is expected to resume the computation (usually by invoking the continuation inside the effect), returning a new state that run will continue to process.

Examples:
> (struct read-env effect-desc (var-name))
> (define my-computation
    (effect (read-env "HOME")
            (lambda (path) (return (string-append "Home is: " path)))))
> (run my-computation
       (lambda (eff)
         (match eff
           [(effect (read-env var) k)
  
            (k "/usr/home/racket-user")])))

"Home is: /usr/home/racket-user"
