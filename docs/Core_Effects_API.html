1 Core Effects API🔗

The core API defines the structure of effectful computations and the machinery to execute them.

struct

(struct pure (value)
    #:extra-constructor-name make-pure
    #:transparent)
  value : any/c
Represents a computation that has completed successfully with a result value. This is the base case of the free monad.

struct

(struct impure (description k)
    #:extra-constructor-name make-impure
    #:transparent)
  description : any/c
  k : (-> any/c any/c)
Represents a suspended computation waiting on an effect.

  • description: The payload describing the requested effect.

  • k: The continuation function. It accepts the result of the effect and returns the next step of the computation.

procedure

(free? m) → boolean?

  m : any/c
Returns #t if m is either a pure or impure value, #f otherwise. This is a type-checking predicate for the free monad.

Examples:
> (free? (return 42))

#t

> (free? (perform 'effect))

#t

> (free? 42)

#f

procedure

(return v) → pure?

  v : any/c
Lifts a raw value v into the effect context. This is equivalent to constructing (pure v).

Example:
> (return 42)

(pure 42)

procedure

(perform desc) → impure?

  desc : any/c
Creates an impure effect descriptor. It takes a description desc and wraps it in an impure structure with the return function as the default continuation. This is used to request an effect that should be handled by an effect handler.

Examples:
> (perform 'read-value)

(impure 'read-value #<procedure:return>)

> (perform (list 'http-get "http://example.com"))

(impure '(http-get "http://example.com") #<procedure:return>)

procedure

(>>= m f) → (or/c pure? impure?)

  m : (or/c pure? impure?)
  f : (-> any/c (or/c pure? impure?))
Sequences two effectful computations. It applies the function f to the result of m once m produces a value.

If m is a pure value, f is applied immediately. If m is an impure, the binding is pushed down into the continuation k, creating a new computation that pauses for the same effect but runs f after the original continuation completes.

Examples:
> (struct increment ())
> (>>= (return 10)
       (lambda (x)
         (impure (increment) (lambda (res) (return (+ x res))))))

(impure #<increment> #<procedure>)

procedure

(run m handle) → any/c

  m : (or/c pure? impure?)
  handle : (-> any/c (-> any/c (or/c pure? impure?)) (or/c pure? impure?))
Executes an effectful computation m by repeatedly resolving effects using the provided handle function.

When run encounters a pure value, it returns the unwrapped value. When it encounters an impure, it extracts the effect description and continuation, then passes them to handle as separate arguments. The handler can process the effect and resume the computation by calling the continuation k, or it can return a new computation to continue execution.

Examples:
> (struct read-env (var-name))
> (define my-computation
    (do-m [path <- (perform (read-env "HOME"))]
        (return (string-append "Home is: " path))))
> (run my-computation
       (lambda (eff k)
         (match eff
           [(read-env var)
  
            (k "/usr/home/racket-user")])))

"Home is: /usr/home/racket-user"
